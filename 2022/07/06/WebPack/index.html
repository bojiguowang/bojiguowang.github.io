Webpack基础笔记

开发模式

1、webpack打包后的文件叫做bundle

2、script标签引入js代码，需要加属性type=”module”，否则浏览器会报错，因为ES6模块化浏览器并不识别，除此之外还可以使用webpack

3、npm i webpack webpack-cli -D 安装webpack 然后使用webpack打包（没写配置文件的情况下），命令为npx webpack index.js – =development,script引入打包后的js文件就不会报错。

4、webpack本身功能极少，只能处理js资源，遇到css或者其他资源就会报错。

5、webpack配置：

    entry(入口)：知识webpack从哪个文佳开始打包
    output（输出）：打包后的文件放哪
    loader（加载器）：webpack本身只能解析js、json资源，其他资源需要借助loader，才能解析
    plugins（插件）：扩展webpack功能
    mode（模式）：两种  开发模式：development   生产模式：production

插件在配置的时候都需要引入

6、webpack配置文件名字一定为：webpack.config.js ，在写了配置文件的情况下，打包命令为npx webpack

内容格式一般是：
    const path = require("path")
    module.exports = {
    // 入口，相对路径
       entry: "./src/main.js",//所有文件必须在入口文件中引入
       // 输出
    output: {
    // 文件输出路径，相对路径，__dirname代表当前文件的文件目录
    path: path.resolve(__dirname, "dist"),
    // 入口文件打包的文件名字
    filename: "main.js"
    },
    module: {
    rules: [
        // loader的位置
        ]
    },
    // 插件
    plugins: {

    },
    // 模式
    mode: "development"
}

7、处理样式资源

1、处理CSS资源

        安装：npm i css-loader style-loader -D
        代码：
        // 处理CSS样式资源
        {
            // 用来匹配 .css 结尾的文件
            test: /\.css$/,
            // use 数组里面 Loader 执行顺序是从下到上
            use: [
                "style-loader", //将js中的CSS文件创建style标签添加到html文件中生效
                "css-loader"//将CSS资源编译成commonjs的模块到js中
            ],
        },

2、处理less资源

        安装：npm i less-loader less -D
        代码：
        {
            test: /\.less$/,
            use: [
                "style-loader", 
                "css-loader", 
                "less-loader"//将less编译成css文件
            ],
        },

3、处理Sass资源

        安装：npm i sass sass-loader -D
        代码：
        {
            test: /\.s[ac]ss$/,
            use: [
                "style-loader", 
                "css-loader", 
                "sass-loader"//将Sass编译成css文件
            ],
        },

4、处理Styl资源

    安装：npm i stylus-loader -D
    代码：
    {
            test: /\.styl$/,
            use: [
                "style-loader",
                "css-loader", 
                "stylus-loader"//将stylus编译成css文件
            ],
        },

8、处理图片资源（没成功，无论是base64处理，还是打包路径）

    webpack4处理图片资源使用file-loader和url-loader，在webpack5中这两个loader已经内置到webpack里了，所以不用在下载了，但是我们有时候需要对图片进行优化，对小于100k的图片转换成base64格式，就是一串编码，减少请求数量，但是会增加一点内存（图片越大，增加的内存越多）。
    不用下载，加入以下代码：
    // 优化图片资源
        {
            test: /\.(png|jpe?g|gif|webp|jpg)$/,
            type: "asset",
            parser: {
                dataUrlCondition: {
                    maxSize: 100 * 1024 // 小于10kb的图片会被base64处理
                }
            },
            //处理图片打包路径
            generator: {
                // 将图片文件输出到 static/imgs 目录中
                // 将图片文件命名 [hash:8][ext][query]
                // [hash:8]: hash值取前8位
                // [ext]: 使用之前的文件扩展名
                // [query]: 添加之前的query参数
                filename: "static/imgs/[hash:8][ext][query]",
              },
        },

9、处理字体图标资源

    字以图标资源和上面的图片资源一样，只要正常引入图标和样式，显示就没有问题，就是输出路径的问题，和上面的也差不多
    {
            test: /\.(ttf|woff2|woff)$/,
            type: "asset/resource",
            generator: {
                filename: "static/media/[hash:8][ext][query]",
              },
        },

这里type如果是asset就是要转化成base64，如果是asset/resource就是原封不动的输出。

10、处理其他资源

    就在上面处理图标资源的test加上要处理的文件后缀，让其打包到同一文件夹下，即可，没有其他要处理的。

11、处理js资源

    处理js格式问题，使用eslint，处理js兼容性问题使用babel。

1、ESlint（插件）

    ESlint最重要的就是配置文件,配置文件有两种方式:
            在根目录下创建 .eslintrc.js文件，
            在package.json文件的eslintConfig配置项下进行配置，后面这种方式不需要新建文件.
    配置文件 .eslintrc.js代码大致结构：
            module.exports = {
        	// 解析选项
        	parserOptions: {
                ecmaVersion: 6, // ES 语法版本
                sourceType: "module", // ES 模块化
                ecmaFeatures: { // ES 其他特性
                    jsx: true // 如果是 React 项目，就需要开启 jsx 语法
                }
            },
            //配置环境问题
            env: {
                node: true,//启动node中全局变量
                browser: true
            },
            // 具体检查规则，"off"或0代表关闭规则,"warn"或1代表警告级别,"error"或2代表错误级别
            rules: {
                "no-var":2,//不能使用var定义变量
            },
            // 继承其他规则
            extends: ["eslint:recommended"],//eslint:recommended是官方推荐的规则
            // 其他规则详见：https://eslint.bootcss.com/docs/user-guide/configuring
        };
    在webpack配置文件中进行配置：
            下载：npm i eslint-webpack-plugin eslint -D
            配置：注意是在插件地方进行配置 
        plugins: [
            new ESLintWebpackPlugin({
              // 指定检查文件的根目录
              context: path.resolve(__dirname, "src"),
                }),
            ],

2、babel

    babel处理兼容性问题
    下载：npm i babel-loader @babel/core @babel/preset-env -D

根目录新建文件 babel.config.js，内容如下： 

    module.exports = {
      // 智能预设，能够编译es6语法为低级的语法
      presets: ["@babel/preset-env"],
    };

webpack.config.js配置，在loader配置项下配置
{
test: /.js$/,
exclude: /node_modules/, // 排除node_modules代码不编译
loader: “babel-loader”,
},

12、处理html资源

    html文件中需要手动引入js文件，安装npm i html-webpack-plugin -D插件并配置插件可以实现自动引入

13、自动化

    每次更改后都需要重新打包，很麻烦，所以下载npm i webpack-dev-server -D并配置后，打包使用命令 npx webpack serve就会开启一个开发服务器，每次更新保存后都会帮你自动打包

开发服务器不会输出资源，是在内存中编译打包，也就是说就算你把dist目录删了，页面还是有效果的。

生产模式

生产模式就是部署上线，新建一个文件夹，用来放开发模式下的webpack配置文件webpack.dev.js和生产模式下的配置文件webpack.prod.js
两个配置文件的区别：

    开发模式不需要输出文件
    生产模式不需要开发服务器
    mode模式记得更改
    两者启动命令不同
        dev：npx webpack serve --config ./webpack/webpack.dev.js
        build：npx webpack --config ./webpack/webpack.prod.js
        可以在package.json中进行配置启动命令
            // package.json
                {
                  "scripts": {
                    "start": "npm run dev",
                    "dev": "npx webpack serve --config ./config/webpack.dev.js",
                    "build": "npx webpack --config ./config/webpack.prod.js"
                  }
                }
        以后启动指令：
        开发模式：npm start 或 npm run dev
        生产模式：npm run build

1、CSS处理

    css 文件目前被打包到 js 文件中，当 js 文件加载时，会创建一个 style 标签来生成样式，会有延迟，造成闪屏，所以我们需要把CSS文件单独打包，并通过link标签加载，性能更好.
    下载：npm i mini-css-extract-plugin -D
    安装完成后，需要将前面style-loader的位置替换成MiniCssExtractPlugin.loader，还需要在插件位置进行引入配置。
    
    css压缩
    下载：npm i css-minimizer-webpack-plugin -D
    在插件位置引入并进行配置
